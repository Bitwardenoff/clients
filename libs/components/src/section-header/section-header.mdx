# Section Header

The `<bit-section-header>` component is meant to be used in combination with the `<bit-section>`
component.

When placed inside of a section with a `bit-card` or `bit-item` as the immediate next sibling (or
nested in the immediate next sibling), the section header will automatically apply bottom padding.

```html
<bit-section>
  <bit-section-header>
    <h2 bitTypography="h6" slot="title">I'm a section header</h2>
    <button bitIconButton="bwi-star" size="small" slot="end"></button>
  </bit-section-header>
  <bit-card>
    <h3 bitTypography="h3">I'm card content</h3>
  </bit-card>
</bit-section>
```

If placed inside of a section without a `bit-card` or `bit-item`, or with a `bit-card`/`bit-item`
that is not a descendant of the immediate next sibling, the bottom padding is not applied.

## Content Slots

`bit-section-header` contains the following slots to help position the content:

| Slot           | Description                     |
| -------------- | ------------------------------- |
| `slot="title"` | title text of the header        |
| `slot="end"`   | placed at the end of the header |

### Title slot

The title should be a `bitTypography` element, usually an `h2` styled as an `h6`.

<Canvas>
  <Story of={stories.OnlyTitle} />
</Canvas>

Title suffixes (typically an icon or icon button) can be passed in within the `title` slot. A gap is
automatically applied between `title` slot children.
